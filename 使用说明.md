## 项目概览

**SAISfold** 是本次 SAIS RNA-蛋白复合物结构预测竞赛的 **Baseline 2（SAIfold）**，核心基于 **DyneTrion-Multimer** 框架，采用 **SE(3) 扩散模型** 预测 RNA-蛋白复合物的三维结构。

### 核心技术路线

```
输入: RNA序列 + 蛋白质序列
        ↓
  ┌─────────────────────────┐
  │  Transformer 编码器       │  ← 序列特征提取 (残基级表示 + 成对相互作用特征)
  └───────────┬─────────────┘
              ↓
  ┌─────────────────────────┐
  │  SE(3) 等变图神经网络      │  ← 三维几何关系建模 (旋转平移不变性)
  │  (IPA - Invariant Point  │
  │   Attention)             │
  └───────────┬─────────────┘
              ↓
  ┌─────────────────────────┐
  │  扩散生成过程 (Diffusion)  │  ← 对三维坐标逐步去噪，生成稳定复合物结构
  │  前向加噪 → 反向去噪       │
  └───────────┬─────────────┘
              ↓
  输出: RNA-蛋白复合物三维原子坐标 (mmCIF/PDB)
```

### 与 Baseline 1 (Protenix) 的区别

| 特性     | Protenix (Baseline 1)      | SAISfold (Baseline 2) |
| -------- | -------------------------- | --------------------- |
| 核心方法 | 坐标扩散 Transformer (DiT) | SE(3) 等变扩散 + IPA  |
| 重点     | 端到端坐标生成             | 序列-结构联合建模     |
| 复杂度   | 较高，完整 Protenix 框架   | 较低，模块化设计      |
| 训练模式 | 统一训练                   | 3种可选训练模式       |
| RNA处理  | 通用生物分子               | 专门适配RNA骨架       |

**关键理解**: 这个仓库由 **多个相对独立的模块** 组合而成，并非单一统一框架：

- **DyneTrion** 是核心扩散模型（来自蛋白质动力学预测领域）
- **RhoFold** 是 RNA 结构预测工具（完整独立项目）
- **OpenFold** 是 AlphaFold2 的开源复现（提供工具函数）
- **Protenix pipeline** 提供数据特征化
- **examples/** 是将这些模块串联起来的"胶水代码"

### 三种预处理模式

```
预处理方式
├── 方式A: 带MSA (需要大型数据库, 最准确)
│   └── 需要: uniref100, mmseqs2, uniclust30, pdb_mmseqs
│
├── 方式B: 不带MSA (单序列, 快速实验) ⭐ 推荐入门
│   └── 使用虚拟MSA/模板特征
│
└── 方式C: 批量预处理 (从mmCIF归档文件)
    └── 用于大规模训练数据生成
```

### 单个结构预处理（方式B - 推荐入门）

```

python examples/preprocess_pdb_protenix.py ^
    --pdb_path data/9dcf.cif ^
    --output_path overfitting_data/9DCF/features.pt

# 从 mmCIF 文件预处理
python examples/preprocess_pdb_protenix.py ^
    --pdb_path data/your_structure.pdb ^
    --output_path features/your_structure.pt
```

**输出 `.pt` 文件内容**：

```

├── rigids_0              # 骨架刚体坐标

│   └── RNA: C4'-C1'-P 三原子定义局部坐标系

├── torsion_angles_mask     # 扭角掩码
└── input_feature_dict      # Protenix兼容的token/atom特征
    ├── token_residue_type   # token残基类型
    ├── token_entity_type    # token实体类型 (蛋白/RNA)
    ├── atom37_positions     # atom37坐标
    └── atom37_mask          # atom37掩码
```

## 训练详解

### 三种训练模式对比



| 特性     | 模式A: 完整DyneTrion           | 模式B: 端到端                           | 模式C: Multimer风格 ⭐                   |
| -------- | ------------------------------ | --------------------------------------- | --------------------------------------- |
| 入口脚本 | `DyneTrion/train_DyneTrion.py` | `examples/train_end_to_end_from_pdb.py` | `examples/train_9dcf_multimer_style.py` |
| 输入     | 预处理的NPZ+嵌入               | 原始PDB文件                             | 预处理特征                              |
| 模型     | diffusion_4d_network_dynamic   | end_to_end_scorenet                     | score_based_ipa (IPA only)              |
| 复杂度   | ⭐⭐⭐⭐⭐                          | ⭐⭐⭐                                     | ⭐⭐                                      |
| 稳定性   | 低 (需要调参)                  | 中                                      | 高 (推荐入门)                           |
| 多帧     | ✅ 支持                         | ❌ 单帧                                  | ❌ 单帧                                  |
| 成对表示 | PairFormer                     | 无                                      | IPA                                     |
| 适合     | 完整研究                       | 快速实验                                | 单结构过拟合/小规模训练                 |

### ⭐ 模式C: Multimer风格训练（推荐入门）

这是**最简单、最稳定**的方式，适合快速上手。

```
# 直接运行 (使用内置的9DCF数据)
python examples/train_9dcf_multimer_style.py
```

**该脚本的核心流程**：

1. 加载预处理好的特征数据
2. 使用 `DyneTrionScoreNet` (仅IPA，无PairFormer)
3. SE(3)扩散训练: 前向加噪 → 网络预测去噪 → 计算损失
4. 保存最佳模型到 `outputs/9dcf_multimer_style/best_model.pth`

**预期结果**: 9DCF复合物(476残基) 在约5000 epoch后达到 ~3.5 Å CA/C4’ RMSD

**自定义训练** (编辑 `train_9dcf_multimer_style.py`)：

```
# 修改以下关键参数:
# 1. 数据路径
data_path = "overfitting_data/YOUR_COMPLEX/features.pt"
# 2. 输出目录
output_dir = "outputs/your_experiment"
# 3. 训练超参
num_epochs = 5000
learning_rate = 1e-4
batch_size = 1
noise_schedule = "lognormal"  # 噪声调度
num_steps = 50               # 扩散步数
```

### ⭐ Multimer风格推理（推荐）

```
python examples/inference_9dcf_multimer_style.py
```

**核心流程**：

1. 加载训练好的模型权重
2. 从纯噪声开始 (或从加噪的真实结构开始)
3. 50步反向扩散去噪
4. 在多个噪声水平下评估直接去噪
5. 输出对齐后的PDB文件

**输出文件**：

```
outputs/9dcf_multimer_style/
├── gt.pdb                    # Ground truth 结构
├── pred_aligned.pdb           # 预测结构 (与GT对齐)
└── sampling_results_9dcf.npz  # 采样结果数据
```


**SE(3)扩散过程详解**:

```
前向过程 (加噪):
  X_0 (真实结构) → X_1 → X_2 → ... → X_T (纯噪声)
  每步: R_t = R_0 * exp(σ_t * ε_rot)    (旋转噪声)
        T_t = T_0 + σ_t * ε_trans         (平移噪声)

反向过程 (去噪):
  X_T (噪声) → X_{T-1} → ... → X_0 (预测结构)
  每步: 网络预测 ε → 使用朗之万动力学采样
```

## 常见问题与解决方案

### `ValueError: not enough values to unpack`

**原因**: 推理时张量维度不匹配
**解决**: 检查推理脚本中是否有过早的 `squeeze` 操作

### 训练出现 NaN 损失

**解决方案**:

1. 关闭辅助损失: `torsion_loss_weight=0, bb_atom_loss_weight=0, dist_mat_loss_weight=0`
2. 添加梯度裁剪: `clip_grad_norm=1.0`
3. 降低学习率
4. 改用Multimer风格训练 (更稳定)

### RNA原子在PDB中看起来不正确

**检查**: 确保数据预处理中RNA骨架原子索引正确:

- atom37 index 0 → C4’
- 原子顺序必须与 `rna_default_frames.npy` 一致

### Win11 特定问题

- **CUDA out of memory**: 减小 `batch_size` 和 `crop_size`
- **deepspeed 安装失败**: 使用 `pip install deepspeed` 或跳过 (非多GPU不需要)
- **路径问题**: Win11 使用 `\` 作为路径分隔符，注意转义

## 关键概念速查表



| 概念              | 解释                                                         |
| ----------------- | ------------------------------------------------------------ |
| **SE(3)**         | 三维欧几里得群的旋转+平移变换，物理结构具有SE(3)等变性       |
| **IPA**           | Invariant Point Attention，等变注意力机制，保证旋转平移不变性 |
| **扩散模型**      | 生成模型，通过前向加噪+反向去噪生成新结构                    |
| **rigid body**    | 刚体表示，由旋转矩阵R和平移向量t组成                         |
| **atom14/atom37** | OpenFold定义的原子表示方式，14为骨架+CB，37为所有重原子      |
| **MSA**           | 多序列比对，提供进化信息，有助于结构预测                     |
| **PairFormer**    | 成对表示更新模块，计算残基间的相互作用                       |
| **EvoFormer**     | 进化信息处理模块 (来自AlphaFold2)                            |
| **C4’-C1’-P**     | RNA骨架三原子，用于定义RNA局部坐标系                         |
| **N-CA-C**        | 蛋白质骨架三原子，用于定义蛋白质局部坐标系                   |
| **lDDT**          | 局部距离差异检验，评估局部结构准确度                         |
| **TM-score**      | 模板建模分数，评估整体结构相似度                             |
| **iRMSD**         | 界面RMSD，评估RNA-蛋白界面的预测精度                         |

> 📌 **总结**: sais-SAISfold 是一个**模块化、多模式**的 RNA-蛋白复合物结构预测框架。建议入门路径：**先用模式C (Multimer风格) 跑通完整流程 → 理解数据预处理 → 尝试模式B端到端 → 最后挑战模式A完整模型**。逐步改进时优先关注 `src/model/score_based_ipa.py` 和 `src/data/se3_diffuser.py` 这两个核心文件。
